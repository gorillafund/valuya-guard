# Protocol Drift Audit (2026-02-25)

Scope audited:

- all adapter packages in `packages/*`
- agent SDK routes and wire calls
- docs and RFCs in repository

## Findings

1. OpenAPI source of truth missing
- No `openapi/` contract file existed in repo.
- Result: adapters and docs cannot be validated against one machine-readable contract.

2. Contract docs fragmented
- RFCs contain multiple protocol fragments, but no short canonical runtime contract for all adapter paths.

3. `prepare` request shape drift risk
- RFC and implementation history include both raw body and wrapped `{ draft: ... }` patterns.
- Canonicalized to wrapper shape; raw body documented as legacy compatibility only.

4. Subject header transition underway
- Canonical header `X-Valuya-Subject-Id` is used.
- Legacy split headers still sent by adapters (`X-Valuya-Subject-Type`, `X-Valuya-Subject-Id-Raw`).

5. Docs entrypoint gaps
- `docs/quickstart.md` is empty.
- `docs/payment-flows.md` is empty.
- This blocks 10-20 minute integration path currently.

6. Contract tests mostly missing
- Only one protocol-related test found: `packages/cli/src/tests/invokeV1.test.ts`.
- No cross-adapter contract suite validating allow/deny/expiry/idempotency behavior.

## Phase 0 alignment actions applied

- Added canonical contract doc: `docs/protocol/canonical.md`
- Added OpenAPI spec: `openapi/v2.yaml`

## Required next edits (Phase 0 completion)

1. Add contract test suite (golden cases):
- allow response
- deny -> checkout payload
- expiry handling
- idempotency for checkout + verify

2. Make docs link to OpenAPI download and canonical contract.

3. Ensure backend/docs publish path for OpenAPI is stable and versioned.
